List of Trajectory Module

Several profile types are available in the trajectory layer. The choice of profile type depends on the desired application. The profile types are:

  • Linear
  • Trapezoidal
  • Sinusodal
  • Position
  • S-curve
  • Homing

List of Linear motor trajectory module parameters

The changeable parameters that are used in linear motor trajectory module and their initial values are listed below.

Table 1 Configuration Options (rm_motor_trajectory_linear)

Options Descriptions
General Customer wants to use their own Trajectory Customised trajectory profile
General Bypass Mode Enable or disable for special trajectories to be managed by the algorithm
General Force alignment Enable or disable to force reference values to align values
General Outer control period [sec] Outer loop control period
interrupt Callback User callback function. If NULL is given, callback is disabled.

Detail of Inverter Parameters

Customer wants to use their own Trajectory

To enable the trajectory feature, you need to set this flag to true after adding the module. The default is false. To prevent malfunctions when the power is turned on, the speed of the target params in the app sample is set to 0. You will also need to correct the speed in order for it to work.

Bypass Mode

set bypass mode when some special trajectories are managed directly by the algorithm (example: motor startup of a specific Sensorless control)

Force alignment

When true, forces reference to align_value. These can be forced by setting the align_value and force_align flag through the Target Set API.

Outer control period

Set the loop period of the speed control period. This value is set automatically. Setting is required when the control period is special, such as for single-shunt operation. The default setting is fine.

interrupt

Set callback function. The callback function called by the algorithm layer to get the next reference value.
